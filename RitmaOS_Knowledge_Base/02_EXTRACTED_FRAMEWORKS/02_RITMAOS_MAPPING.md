# Mapping Course Ideas to RitmaOS

| Course idea | RitmaOS interpretation |
|---|---|
| Folder structure changes AI behavior | Workspace files become persistent business context |
| Prompting framework | Task-level operating instructions |
| AI is the next unreliable layer | RitmaOS is the reliability layer |
| Mad Libs / template pattern | Workflows become typed slots filled by files and prompts |
| Value moves up abstraction layers | RitmaOS owns the layer above generic AI tools |
| Orchestration > intelligence | RitmaOS routes AI, files, tools, and humans |
| 60/30/10 | Product should be mostly structure, routing, and data flow |
| Spec-driven production | Workflows should start from specs, not random prompts |
| Agents are sequenced prompts + software | MVP can be a filesystem workflow before becoming software |

## Product thesis

RitmaOS turns a business workflow into an AI-readable filesystem.

## Buyer-facing language

Most businesses are using AI like a chat box. RitmaOS turns it into an operating system for work.

## Builder-facing language

RitmaOS is a filesystem-based orchestration layer for AI-powered work.
