# Commit Guidelines

- Use small, purposeful commits.
- Keep raw source additions separate from synthesis.
- Keep structure changes separate from product-writing changes when practical.
- Prefer additive changes.
- Ask before destructive changes.
- Update indexes and `PACKAGE_MANIFEST.md` in the same commit as structural changes.

Recommended commit groups:

1. Raw source addition.
2. Framework extraction.
3. Product spec update.
4. Decision log update.
5. Repository architecture or operations update.
