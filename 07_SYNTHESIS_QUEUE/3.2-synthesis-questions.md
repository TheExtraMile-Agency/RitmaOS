# Module 3.2 Synthesis Questions

Source raw module: `02_RAW_MODULES/3.2-customizing-for-your-use-case.md`

## Questions

- What should be RitmaOS’s first supported archetype: content creator, freelancer/consultant, developer, or small business/agency?
- Should RitmaOS generate fixed workspace names or ask users to name their own work modes?
- How many starter workspaces should the MVP create by default?
- Should every generated workspace include a `CONTEXT.md` file from day one?
- Should skills/tool mappings be included in the MVP or delayed?
- How should RitmaOS prevent context bleed between clients, projects, or work modes?
- Should client folders be treated as workspace instances generated from a reusable client template?
- Should context files be explicitly treated as living documents with periodic review prompts?
- What naming conventions should be universal versus archetype-specific?
- How much customization should happen during onboarding versus after the first real use?

## Synthesis Direction

Module 3.2 should be synthesized with 3.1 because 3.1 defines the base architecture and 3.2 defines how that architecture becomes adaptable across real user types.
