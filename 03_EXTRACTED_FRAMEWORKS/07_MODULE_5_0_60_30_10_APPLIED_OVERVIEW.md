## Core Framework

Module 5.0 shifts the 60/30/10 ratio from explanatory theory into applied business and delivery contexts.

The reusable frame is:

```text
60% infrastructure -> 30% orchestration -> 10% AI
```

The overview positions the ratio as a practical operating model for:

- building AI-enabled software,
- pricing and contracting client work,
- structuring multi-stream partnerships,
- converting training into custom builds,
- mapping a solo operator's leverage system.

## Series Structure

The applied series has three entry points:

| Lesson | Primary Use Case | Framework Face |
|---|---|---|
| Lesson 1: The Build / VigilOre | Software and technical project scoping | System architecture determines build scope, moat, pricing, and contract structure. |
| Lesson 2: The Partnership / NLP Logix | Business development, freelancing, and client relationships | Partnership structure compounds across revenue streams and operating loops. |
| Lesson 3: The System Underneath | Strategic view of training, adoption, and leverage | Training generates structured evidence, which becomes specs, pilots, and repeatable delivery. |

The order can vary by user need. Technical builders start with Lesson 1, business builders start with Lesson 2, and system thinkers start with Lesson 3.

## Applied Layer Translation

The lesson frames the same ratio across multiple rooms:

| Context | 60% Infrastructure | 30% Orchestration | 10% AI |
|---|---|---|---|
| VigilOre build | Parser, scorer, rate limiter, deterministic logic, compliance platform structure | Multi-agent flow, branching interview logic, workflow coordination | Short AI calls inside each agent |
| VigilOre business model | Pricing, contract terms, IP protection, month-to-month structure | How architecture supports offer design and risk allocation | AI-assisted compliance outcome |
| NLP Logix partnership | Contract infrastructure, invoicing, course library | Revenue streams feeding each other into a professional flywheel | AI used in training and delivery |
| Training funnel | Company training, exercises, data capture | Training-to-spec-to-build conversion | AI recommendations and delivery |
| Creator/operator system | Content production, folder architecture, rendering pipelines | Content routing, partnerships, training-to-pilot conversion | Generation and analysis calls |

## Pattern Skill Over Tool Skill

Module 5.0 introduces a durable distinction:

```text
tool skill decays; pattern skill survives tool changes
```

The 60/30/10 model is presented as a pattern skill. It should transfer across tools, clients, industries, and delivery models because it teaches where AI belongs rather than which AI button to press.

## Exercise Pattern

Every lesson includes a course companion exercise that applies the framework to the user's real work.

The exercise pattern is:

```text
real role -> real bottleneck -> layer audit -> applied redesign
```

This matters for RitmaOS because the framework becomes valuable when it turns user-specific work into structured layer maps, not when it remains a generic concept.

## RitmaOS Extraction

Module 5.0 suggests RitmaOS should treat 60/30/10 as both:

- a diagnostic framework for current workflows,
- a design framework for future software, services, partnerships, and training funnels.

The product should help users map work into layers, identify where time is being spent in the wrong layer, and turn those observations into workspace structure, service design, and implementation specs.
