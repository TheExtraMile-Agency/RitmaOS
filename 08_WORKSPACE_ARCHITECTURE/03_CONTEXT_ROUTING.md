# Context Routing

Context routing means placing information where the next agent, model, or human can find and use it.

Working ideas:

- Raw source stays separate from interpretation.
- Stage-specific context belongs near the work stage that needs it.
- Shared context belongs in a shared folder.
- Decisions should be durable and easy to cite.
- Outputs should be reusable as future inputs.

The filesystem acts as a plain-English routing layer.
