# Folder Schema

RitmaOS uses folders as intentional context boundaries.

Working schema:

- `intake/` stores the request, source material, and constraints.
- `analysis/` stores interpretation and decomposition.
- `draft/` stores generated work in progress.
- `review/` stores critique, checks, and revision notes.
- `delivery/` stores final outputs.
- `memory/` stores reusable decisions and lessons learned.

The schema should stay understandable to both humans and AI agents.
